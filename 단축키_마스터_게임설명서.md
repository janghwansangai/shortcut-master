# 🎮 단축키 마스터 (Shortcut Master) - 기술 명세 및 시스템 아키텍처 정의서

본 문서는 "단축키 마스터(Shortcut Master)" 프로젝트의 전반적인 게임 로직, 시스템 아키텍처, 네트워크 동기화 방식 및 개발 환경을 상세히 기술한 공식 기술 문서입니다.
> **공식 기술 문서 (Technical Specification)**: 프로젝트의 설계도 역할을 합니다. 코드를 작성하기 전, 시스템이 어떻게 작동할지 개발자들끼리 약속하는 문서입니다.

---

## 1. 프로젝트 개요 (Project Overview)

"단축키 마스터"는 고전적인 버블 슈터(Bubble Shooter) 게임 메커니즘에 키보드 단축키(Shortcuts) 학습 시스템을 결합한 웹 기반 멀티플레이어 아케이드 게임입니다.
> **웹 기반 아케이드 게임 (Web-based Arcade Game)**: 별도의 설치 없이 인터넷 브라우저만 켜면 바로 작동하는 게임입니다. JavaScript 언어가 브라우저 위에서 실행되며 동작합니다.

사용자는 화면 하단에서 단축키 키(Key)가 각인된 버블을 발사하여 상단의 버블과 결합시키며, 동일한 단축키로 이루어진 유효한 조합(예: `Ctrl` + `C`)이 완성되거나 동일한 키가 3개 이상 연결될 경우 버블이 파괴되며 점수를 획득합니다.
> **조합 로직 (Combination Logic)**: 배열이나 세트에 특정 키 조합이 들어있는지 검사하는 프로그래밍 기법입니다.
> ```javascript
> // 예시: 배열에 특정 단축키가 존재하는지 확인하는 로직
> const poppedKeys = ['Ctrl', 'C'];
> const isMissionClear = poppedKeys.includes('Ctrl') && poppedKeys.includes('C');
> if (isMissionClear) score *= 5;
> ```

## 2. 시스템 아키텍처 및 기술 스택 (Tech Stack)

### 2.1 Frontend (Client)
- **Framework**: React 18 (Vite 기반 빌드)
  > **Framework (프레임워크) 및 React**: 프레임워크란 개발을 쉽게 할 수 있도록 뼈대와 규칙을 제공하는 도구입니다. React는 화면(UI)을 부품(Component) 단위로 쪼개어 조립할 수 있게 해주는 가장 인기 있는 도구입니다.
  > ```jsx
  > // React를 이용해 화면의 부품(Component)을 만드는 예시
  > function PlayerScore({ score }) {
  >   return <div className="score">현재 점수: {score}</div>;
  > }
  > ```
- **Language**: TypeScript (엄격한 타입 체크를 통한 안정성 확보)
  > **TypeScript (타입스크립트)**: JavaScript에 '타입(형태)'을 강제하여 에러를 미리 막아주는 언어입니다. 숫자가 들어갈 곳에 문자가 들어가면 코드 실행 전에 빨간 줄을 띄워 알려줍니다.
  > ```typescript
  > // TypeScript 예시: score는 반드시 숫자여야 함
  > function addScore(current: number, points: number): number {
  >   return current + points;
  > }
  > // addScore(100, "50"); // 에러 발생!
  > ```
- **Styling**: Tailwind CSS (반응형 UI 및 유틸리티 클래스 기반 스타일링)
  > **Tailwind CSS**: HTML 요소 안에 직접 클래스 이름을 넣어 디자인을 꾸미는 도구입니다.
  > ```html
  > <!-- 텍스트는 크게(text-xl), 굵게(font-bold), 글자색은 빨강(text-red-500) -->
  > <div className="text-xl font-bold text-red-500">GAME OVER</div>
  > ```
- **Rendering Engine**: HTML5 Canvas API (게임 렌더링 최적화를 위해 DOM 대신 Canvas 채택)
  > **HTML5 Canvas API**: 일반적인 HTML 태그를 움직이는 대신, 도화지(Canvas) 하나를 깔아놓고 그 위에 픽셀 단위로 그림을 그리는 기술입니다. 매우 빨라서 게임 개발에 필수적입니다.
  > ```javascript
  > // Canvas를 사용해 파란색 원(방울) 그리기
  > const ctx = canvas.getContext('2d');
  > ctx.beginPath();
  > ctx.arc(x, y, radius, 0, Math.PI * 2);
  > ctx.fillStyle = 'blue';
  > ctx.fill();
  > ```
- **State Management**: React Hooks (`useState`, `useEffect`, `useRef`) 기반 자체 상태 관리 및 Socket 상태 연동
  > **React Hooks (`useState` 등)**: React에서 변수(상태)를 저장하고 관리하는 기능입니다. 상태가 변하면 화면이 자동으로 새로고침(Re-render)됩니다.
  > ```jsx
  > // useState 사용 예시
  > import { useState } from 'react';
  > function Counter() {
  >   const [score, setScore] = useState(0); // 초기 점수 0
  >   return <button onClick={() => setScore(score + 10)}>점수 올리기: {score}</button>;
  > }
  > ```
- **Deployment**: Vercel (CI/CD 자동화 및 엣지 네트워크 배포)
  > **Vercel 및 CI/CD**: 내가 짠 코드를 GitHub에 올리기만 하면, 알아서 인터넷(서버)에 배포(적용)해주는 자동화 서비스입니다.

### 2.2 Backend (Server)
- **Runtime**: Node.js
  > **Node.js (런타임)**: 원래 브라우저에서만 돌아가던 JavaScript를, 서버(컴퓨터)에서도 돌아가게 만들어주는 실행 환경입니다.
- **Framework**: Express.js
  > **Express.js**: Node.js 환경에서 서버를 아주 쉽게 만들 수 있도록 도와주는 백엔드 프레임워크입니다.
  > ```javascript
  > // Express 서버 생성 예시
  > const express = require('express');
  > const app = express();
  > app.get('/', (req, res) => res.send('게임 서버 작동 중!'));
  > app.listen(3000);
  > ```
- **WebSocket**: Socket.io (실시간 양방향 통신 및 멀티플레이 동기화)
  > **WebSocket (Socket.io)**: 새로고침 없이도 1초에 수십 번씩 데이터를 주고받는 실시간 통신 기술입니다. 카카오톡이나 멀티플레이 게임에 쓰입니다.
  > ```javascript
  > // Socket.io 통신 예시
  > // 서버가 "공격해!" 라는 메시지를 보내면,
  > io.emit('incoming_attack', { power: 10 });
  > 
  > // 클라이언트가 받아서 처리합니다.
  > socket.on('incoming_attack', (data) => {
  >   console.log(data.power + " 만큼 데미지를 입었습니다!");
  > });
  > ```
- **Deployment**: Render (웹소켓 연결 유지를 위한 Persistent Server 환경 구성)
  > **Persistent Server (영구 서버)**: 게임 대기방처럼 유저들이 계속 연결되어 있어야 하므로, 서버가 잠들지 않고 24시간 내내 켜져 있도록 유지하는 환경입니다.

## 3. 핵심 게임 로직 (Core Game Logic)

### 3.1 렌더링 및 물리 엔진 (Rendering & Physics)
- **Game Loop**: `requestAnimationFrame`을 사용하여 60 FPS 기반의 게임 루프를 구현 (`GameEngine.ts`). 매 프레임마다 버블의 위치 업데이트 및 화면 재렌더링 수행.
  > **requestAnimationFrame (Game Loop)**: 게임 화면을 모니터의 주사율(보통 1초에 60번)에 맞춰 최대한 부드럽게 그려주는 브라우저 내장 함수입니다.
  > ```javascript
  > // 게임 루프 예시
  > function gameLoop() {
  >   updatePositions(); // 1. 방울 위치 계산
  >   drawEverything();  // 2. 화면에 그리기
  >   requestAnimationFrame(gameLoop); // 3. 무한 반복
  > }
  > requestAnimationFrame(gameLoop); // 최초 실행
  > ```
- **충돌 감지 (Collision Detection)**: 유클리디안 거리 공식을 사용하여 발사된 버블의 중심 좌표와 맵에 존재하는 버블들의 중심 좌표 간 거리가 `2 * radius` 이하가 되는 순간을 충돌로 판정.
  > **유클리디안 거리 공식 (Euclidean Distance)**: 피타고라스의 정리를 사용해 2차원 화면에서 두 점(버블) 사이의 거리를 구하는 수학 공식입니다.
  > ```javascript
  > // 두 방울 사이의 거리 계산 코드
  > const dx = bubble2.x - bubble1.x;
  > const dy = bubble2.y - bubble1.y;
  > const distance = Math.sqrt(dx * dx + dy * dy); // 피타고라스 정리
  > if (distance <= radius * 2) {
  >   console.log('충돌!');
  > }
  > ```
- **그리드 스냅 (Grid Snapping)**: 헥사고날(Hexagonal) 엇갈린 그리드 시스템 채택. 충돌 시 가장 가까운 빈 그리드 셀 좌표를 수학적으로 계산하여 버블을 해당 위치에 안착시킴.
  > **헥사고날 그리드 (Hexagonal Grid)**: 벌집 모양의 격자를 뜻합니다. 버블들이 사각형이 아니라 지그재그로 물려있도록 수학적으로 행(Row)마다 X축을 절반씩 빗겨서 계산합니다.

### 3.2 알고리즘 (Algorithms)
- **버블 매칭 알고리즘 (BFS)**:
  버블이 안착하는 순간, 해당 버블을 시작점으로 BFS(너비 우선 탐색) 알고리즘을 수행하여 인접한 동일 키 버블을 그룹화함. 그룹의 크기가 3 이상이거나 특정 단축키 조합을 완성하면 파괴(Pop) 대상 배열에 추가.
  > **BFS (너비 우선 탐색 알고리즘)**: 게임에서 나와 똑같은 색상의 방울이 얼마나 연결되어 있는지 '가장 가까운 곳부터 넓게 퍼져나가며' 찾아내는 길찾기 알고리즘입니다.
  > ```javascript
  > // BFS 로직 구조 예시
  > const queue = [startBubble]; // 탐색을 시작할 방울
  > const visited = new Set();
  > while(queue.length > 0) {
  >   const current = queue.shift(); // 하나 꺼내서
  >   visited.add(current);          // 방문했다고 표시
  >   // 내 주변에 있는 방울을 큐에 다시 넣는다
  >   queue.push(...getNeighbors(current)); 
  > }
  > ```
- **공중에 뜬 버블 감지 알고리즘 (Floating Bubbles Detection)**:
  매칭으로 인해 일부 버블이 파괴된 후, 천장(0번째 행)에 붙어있는 모든 버블을 시작점으로 하여 연결된 모든 버블을 BFS로 탐색. 탐색되지 않은(천장과 연결이 끊어진) 버블들은 중력에 의해 떨어지는(Drop) 효과를 적용 후 배열에서 제거.
  > **플로팅 감지 (Floating Detection)**: 연결된 방울이 터지고 나서, 공중에 둥둥 떠서 '천장'과 연결되지 않은 방울을 찾아내어 밑으로 떨어뜨리는(중력) 시스템입니다.

### 3.3 점수 및 콤보 시스템 (Scoring System)
- **기본 점수**: 파괴된 버블의 개수에 비례하여 점수 부여.
- **콤보 (Combo)**: 연속으로 버블을 파괴할 때마다 콤보 카운터가 증가하며, 점수에 `(1 + 콤보 * 0.1)` 등의 배율 곱(Multiplier) 적용. 실패 시 콤보 초기화.
  > **배율 곱 (Multiplier)**: 점수가 기하급수적으로 늘어나게 하는 수학적 공식입니다.
  > ```javascript
  > // 점수 계산 코드 예시
  > const baseScore = 100;
  > const combo = 3;
  > const finalScore = baseScore * (1 + combo * 0.1); 
  > // 100 * 1.3 = 130점 획득!
  > ```
- **미션 시스템**: 현재 활성화된 "미션(예: 복사 - `Ctrl + C`)"에 해당하는 버블 조합을 파괴할 경우 최종 획득 점수에 **x5 보너스 배율** 적용.
- **3키 조합 보너스**: `Ctrl + Shift + Esc` 등 3개의 키로 이루어진 고난이도 단축키 조합 파괴 시 **x6 보너스 배율** 적용 및 공격 게이지 대폭 상승.

## 4. 멀티플레이어 네트워크 데이터 처리 (Multiplayer Data Flow)

### 4.1 주요 Socket Event 구조
1. **`joinRoom` / `leaveRoom`**
   특정 네임스페이스(방)에 소켓을 조인하고, 다른 클라이언트들에게 신규 유저 진입을 알림.
   > **joinRoom (방 접속 로직)**: 특정한 고유 문자열(예: 'ROOM_123')을 키 값으로 사용하여 유저들을 한 공간에 묶는 로직입니다.

2. **`player_state_update` (Client -> Server)**
   클라이언트가 자신의 캔버스 그리드, 점수, 콤보 상태를 서버로 전송.
   > **player_state_update (상태 업데이트)**: 클라이언트에서 연산이 끝난 내 화면의 최종 결과물을 백엔드(서버)에게 보고하는 패킷(데이터 묶음)입니다.

3. **`room_state_update` (Server -> Client)**
   서버가 방 내부의 모든 플레이어의 최신 상태 목록을 브로드캐스트(Broadcast).
   > **브로드캐스트 (Broadcast)**: 서버가 특정 방에 모여있는 '모든' 유저에게 한 번에 데이터를 쏘아 보내는(방송하는) 네트워크 기술입니다.
